# AGENTS.md

## 🧠 Agent Context Protocols
Warp agents in this workspace operate with the following context layers:

- **Context 7 MCP**: Provides live context from GitHub, Brave, and internal docs. Agents use this to resolve library IDs, fetch documentation, and apply coding standards.
- **Puppeteer MCP**: Enables browser automation and UI testing. Agents can scaffold test flows, generate selectors, and validate DOM interactions.

Agents automatically ingest rules from:
- `AGENTS.md` (this file)
- `WARP.md` (project-wide and subdirectory-specific)
- Global Rules (from Warp Drive > Personal > Rules)

---

## 🚀 Dev Environment Tips
- Use `pnpm dlx turbo run where <project_name>` to jump to a package instead of scanning with `ls`.
- Run `pnpm install --filter <project_name>` to ensure Vite, ESLint, and TypeScript recognize the workspace.
- Create new packages with:  
  `pnpm create vite@latest <project_name> -- --template react-ts`
- Confirm package names inside each `package.json` — ignore the top-level one.

### 🧠 Agent Rule
> Agents should auto-resolve package names using Context 7’s `resolve-library-id` before running install or scaffold commands.

---

## ✅ Testing Instructions
- CI plans live in `.github/workflows`.
- Run full test suite:  
  `pnpm turbo run test --filter <project_name>`  
  or from root: `pnpm test`
- Focused test runs:  
  `pnpm vitest run -t "<test name>"`
- After file moves or import changes:  
  `pnpm lint --filter <project_name>`
- Always fix type and test errors before merging.
- Add/update tests for any code you touch — even if not requested.

### 🧠 Agent Rule
> Agents must validate test coverage and lint status before allowing commit. Use Context 7 to fetch lint configs and Puppeteer MCP to scaffold UI test flows.

---

## 📦 PR Instructions
- Title format: `[<project_name>] <Title>`
- Run `pnpm lint` and `pnpm test` before committing.

### 🧠 Agent Rule
> Agents enforce PR hygiene: title format, lint/test pass, and commit message standards. Use emojis and aliases as defined in Global Rules.

---

## 🧠 Contextual Automation Rules

### Git as Save Point
- Every commit uses aliases, emojis, and format rules.
- Warp prompts “Do you want to save your progress?” after major changes.

### ls vs lolcat
- `ls` aliased to rainbow puke. Broke stuff.
- Rule: Use `\ls` when you need real `ls`.

### Project Spin-Up
- Warp scaffolds repos, renames `master` → `main`, wires GitHub, sets up docs.
- Command: `new project` triggers full boilerplate.

### GAS Projects
- Rules handle `.claspignore`, `@types/google-apps-script`, and deployment.
- Never think about it again.

### Context Stacking
- In Git repos, Warp auto-pulls, runs `git status`, pipes logs through `forgit`.
- Terminal handles hygiene: status, logs, diffs, and commit prompts.

---

## 🧩 Agent Behavior Expectations

- Agents must:
  - Use **Context 7 MCP** to resolve libraries, fetch docs, and apply standards.
  - Use **Puppeteer MCP** to scaffold browser tests and validate UI flows.
  - Respect directory-specific `WARP.md` rules before applying global ones.
  - Avoid destructive commands (`rm -rf`, `kill -9`) unless explicitly confirmed.
  - Scaffold minimal, reproducible examples when asked to fix or refactor.
  - Ask clarifying questions if context is ambiguous or overengineered.
